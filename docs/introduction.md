# Introduction to w-csl

The w-csl is a project that aims to provide a common interface for the different services that are part of the CSL.
The w-csl is divided into two parts: the `CSL_CLIENT` and the `CSL_SERVER`.

The CSL_CLIENT is a client that is used to communicate with the local services that are running on the client's network.
The CSL_SERVER is HTTP to WebSocket server (Proxy Server) that is used to communicate with CSL_CLIENT from the outside of the CSL Platform.
The w-csl is written in Java.
